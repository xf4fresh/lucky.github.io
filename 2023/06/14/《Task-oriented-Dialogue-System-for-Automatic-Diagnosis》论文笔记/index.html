Markov Decision Process Formulation for Automatic Diagnosis

We cast DS as Markov Decision Process (MDP) (Young et al., 2013) and train the dialogue policy via reinforcement learning (Cuayahuitl et al., 2015). An MDP is composed of states, actions, rewards, policy, and transitions.



Learning with DQN

the policy is parameterized with a deep Q-network (DQN) (Mnih et al., 2015), which takes the state as input and outputs for all actions .

A Q-network can be trained by updating the parameters at iteration to reduce the mean squared error between the Q-value computed from the current network and the Q-value obtained from the**_ Bellman equation_** , where is the target network with parameters from some previous iteration.

：reward
：超参

In practice, the behavior distribution is often selected by an -greedy policy that takes an action with probability and selects a random action with probability , which can improve the efficiency of exploration.

：超参

When training the policy, we use a technique known as experience replay. We store the agent’s experiences at each time-step, in a fixed size, queue-like buffer .

In a simulation epoch,

  • the current DQN network is updated multiple times (depending on the batch size
  • and the current size of replay buffer) with different batches drawn randomly from the buffer, while the target DQN network is fixed during the updating of current DQN network.

At the end of each epoch, the target network is replaced by the current network and the current network is evaluated on training set. The buffer will be flushed if the current network performs better than all previous versions.
